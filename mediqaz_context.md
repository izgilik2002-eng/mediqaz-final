# MediQaz — Контекст проекта

---

## Что такое MediQaz

MediQaz — это веб-приложение, работающее как AI-ассистент для врачей: оно в реальном времени транскрибирует разговор между врачом и пациентом через микрофон, а затем автоматически заполняет стандартные медицинские формы Damumed (МИС Казахстана), используя большую языковую модель. Врач получает готовый структурированный медицинский документ сразу по завершении приёма, без ручного ввода.

---

## Целевой пользователь

**Врачи в Казахстане**, работающие в государственных и частных клиниках, подключённых к МИС Damumed. Прежде всего — врачи амбулаторного звена (поликлиники, семейные врачи), у которых высокий поток пациентов и хронически не хватает времени на документацию.

---

## Какую проблему решает

Врачи тратят до 30–40% рабочего времени на заполнение медицинских форм вручную. MediQaz устраняет эту нагрузку:

- **Нет ручного ввода** — форма заполняется автоматически по транскрипции.
- **Нет переключения контекста** — врач не отвлекается от пациента.
- **Нет языкового барьера** — транскрипция понимает русско-казахский микс (code-switching), типичный для казахстанских врачей.
- **Простая выписка для пациента** — LLM автоматически генерирует краткое человекочитаемое резюме без медицинского жаргона.

### AI / Speech
| Технология | Назначение |
|---|---|
| **Deepgram Nova-2** | Real-time STT стриминг через WebSocket, диаризация спикеров (Врач/Пациент) |
| **Groq API (llama-3.3-70b-versatile)** | Сверхбыстрый LLM-инференс для заполнения форм и Magic Edit |

### Frontend
- Чистый **HTML + Vanilla JS + CSS** (без фреймворков), файлы в `/public`
- WebSocket-клиент для передачи аудио-стрима и получения обновлений формы в реальном времени

### DevOps / Инфраструктура
| Технология | Назначение |
|---|---|
| **Docker** (node:20-bullseye-slim) | Контейнеризация, готовый образ для деплоя |
| **docker-compose** | Локальный запуск |
| **Railway** | Облачный деплой (production) |
| **ESLint + Prettier** | Линтинг и форматирование кода |
| **nodemon** | Hot-reload в режиме разработки |

## Что уже сделано и работает

- ✅ **Real-time транскрипция** — аудио стримится по WebSocket → Deepgram Nova-2 → текст обновляется на экране врача в реальном времени
- ✅ **Диаризация спикеров** — автоматическое разделение речи на `Врач` / `Пациент` через Deepgram
- ✅ **Автозаполнение форм** через LLM (Groq llama-3.3-70b) с debounce-логикой и UtteranceEnd триггером
- ✅ **3 формы Damumed** с готовыми промптами:
  - Форма **052/у** — Амбулаторный приём
  - Форма **035/у** — Лист временной нетрудоспособности (больничный)
  - Форма **130/у** — Рецептурный бланк
- ✅ **Простая выписка для пациента** — генерируется автоматически как часть формы (`patient_summary`)
- ✅ **Magic Edit** — врач голосом или текстом корректирует уже заполненную форму
- ✅ **Кастомный промпт на врача** — каждый врач может задать свои правила, аббревиатуры, предпочтения для LLM
- ✅ **Полная аутентификация** — регистрация / вход по email + JWT, изоляция данных по `user_id`
- ✅ **История приёмов** — поиск и просмотр всех сохранённых приёмов врача
- ✅ **Безопасность** — rate limiting на auth, валидация входных данных, bcrypt-хэш паролей
- ✅ **Docker-сборка** — multi-stage оптимизация, production-ready образ
- ✅ **Деплой на Railway** — приложение живёт в облаке

## Что планируется (Roadmap)

### Ближайшие задачи (v1.x)
- [ ] Поддержка казахского языка как основного в транскрипции (Deepgram `language: 'kk'` или мультиязычный режим)
- [ ] Экспорт формы в PDF / DOCX для загрузки в Damumed
- [ ] Интеграция с API Damumed (прямая отправка формы в МИС)
- [ ] Улучшение UI/UX интерфейса (тёмная тема, мобильная адаптация)

### Среднесрочно (v2.0)
- [ ] Автоматический ICD-10 / МКБ-10 кодинг диагноза
- [ ] Добавление новых форм Damumed (стационарные, профосмотры, выписной эпикриз)
- [ ] Веб-панель администратора клиники (управление врачами, статистика приёмов)
- [ ] Мобильное приложение (React Native / PWA)

### Долгосрочно (v3.0+)
- [ ] Self-hosted LLM-деплой (Ollama) для клиник с требованиями HIPAA / конфиденциальности
- [ ] Speaker diarization собственной моделью (без Deepgram) для снижения затрат
- [ ] Аналитика по диагнозам и нагрузке врача
